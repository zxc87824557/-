Sass

Sass 是 CSS的預處理器(preprocessor)，目的在撰寫CSS時減少反覆重工的動作以及更有效率的樣式結構維護，sass 必須進行編譯(complier)或原生的css才能使用，也就是說瀏覽器不懂sass，需經過編譯才能閱讀。

Css 不是程式語言，沒有流程控制、函式這些功能，sass 除了可以使用崁套規則、繼承、混合(mixins)這些功能，還可以使用變數(variables)、函式(function)、流程控制(flow control)等等，所以也可以稱為css的程式語言。

Sass 與 Scss

Sass 分為兩種 sass 與 scss，都稱為sass。不同點如下:

副檔名 年分 語法 說明
.sass 2007年，最早的預處理器 縮排語法 (沒有css大瓜，結尾分號) 語法不能被大眾接受，無法兼容css
.scss 推出時期less之後 塊語法 (寫法與css一致，兼容css) 受到less影響，scss可以相容css，撰寫起來親和彈性，成為普及受歡迎的預處理器

1.變數 (variables)

變數使用時機，當一個植被重複使用兩次以上十，就可以考慮使用

變數可以統一定義網站規範，只需修改數值，不須再css原始黨一一修改

底線(_) 連結線(-) 的使用可以互換。定義變數名稱是 $bg_black，但可以透過 $bg-black 來提取。

2. 崁套(Nested)

崁套雖然好用，不代表所有層級都該寫入崁套裡面，可以考慮使用最少的選擇棄選到位置。

過度崁套編譯後選擇器過長，會影響到層及執行，也會造成css檔案容量過大。

以區塊崁套有個優點，partials的概念讓結構區塊模組化，專案維護時只需針對錯誤找到模組位置修改。

3. Extend (繼承)

繼承可以用css選擇器或%開始命名

使用css選擇器，會被編譯到css，而%佔位符不會被編譯到css。

使用@extend可以養成習慣，統一將其放在樣式的最前面。

相同的繼承樣式會在編譯後會被使用群組選擇器合併在一起‧

4. 計算

可以做加、減、乘、除、餘數的計算

法比較特別，遇到除法計算，養成用括號的習慣

sass的巨算與calc()不同點在於預處理前已經計算好結果，不會造成瀏覽器的負擔，calc()是在文件正被瀏覽器解讀時進行，會消耗瀏覽器資源。

5. Function(函式)

@function 與 @mixin 有點像，都是在做運算處理，兩者差異在於 @function 必須使用@return 來返回結果，而且輸出式一個值，並非是一段 css 樣式，@mixin 則可以使用 @include 產生一段樣式。

8.資料型態

  • 1. number(數字):數字有分有單位跟無單位， ,1,2,13,10px
  • 2. string(字串):有引號字串，"for" "bar" 無引號字串 baz
  • 3. color(顏色): blue, #000000,rgba(0,0,0,0.5)
  • 4. boolean(布林值):true、false
  • 5. null(空值):null
  • 6. list(向陣列):用空格或逗號做為分隔，1.5rem 1rem 0 2rem 或 "arial,sans-serif"
  • 7.map(物件):類似js的物件
